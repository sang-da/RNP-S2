# 06 - Le Playbook de l'Enseignant (Game Master)

Avoir une application fonctionnelle est une chose, savoir l'animer en classe en est une autre. Ce document détaille la routine type d'un enseignant utilisant RNP Manager pour éviter l'épuisement et maximiser l'engagement.

## 1. La Posture du Game Master
Ne soyez plus un "professeur qui note". Soyez :
- **Le Banquier** : Qui réclame les loyers.
- **Le Marché** : Qui impose des pénalités si un livrable est en retard.
- **Le Scénariste** : Qui lance des "Black Ops" pour tester les équipes.

**Règle d'or : Ne débattez jamais.** Si un étudiant se plaint d'avoir perdu 10 VE, répondez : *"C'est le marché. Le rapport d'audit dans votre historique indique un livrable non conforme. Adaptez-vous pour la semaine prochaine."*

## 2. La Routine Hebdomadaire Type

**Lundi Matin : Ouverture du Marché**
- Déblocage des nouvelles missions de la semaine via l'interface Admin.
- Annonce publique s'il y a des modifications ("Le prix du loyer augmente de 10% pour les agences ayant plus de 1000 PiXi").

**Mercredi : Le Mid-Week (Optionnel mais recommandé)**
- Vérification rapide de l'engagement. Si l'IA signale qu'un étudiant n'a pas été actif, ou qu'une agence est au bord de la banqueroute, lancez un "Avertissement Financier" via l'UI.

**Vendredi 23h59 : Fermeture et Prélèvements**
- Limite de dépôt des livrables (Upload). Toute seconde de retard = refus du système ou malus automatique.
- Le Game Master clique sur **"Prise de Loyer"** et **"Paiement des Salaires"**. Le système débite et crédite toutes les agences.

## 3. Gérer la "Banqueroute" et la Frustration
Un étudiant sans PiXi ne peut plus payer son agence, un étudiant viré se retrouve au chômage. **C'est le but.**
Cependant, l'objectif n'est pas de détruire les élèves. 
- **Le sauvetage (Bailout)** : Les agences riches peuvent prêter aux plus pauvres (P2P). Laissez-les négocier entre eux dans la réalité.
- **Les Missions de la Dernière Chance** : Proposez des "bounties" (Missions très dures, très bien payées) pour qu'une équipe puisse se refaire.

## 4. Les Révisions par les Pairs (Evaluation)
A la fin d'un cycle court de mission, activez le "Peer Review".
Ne notez pas vous-même ! Les équipes A évaluent les équipes B. L'algorithme se chargera d'éliminer les notes aberrantes (les trolls qui mettent 0 ou les pactes de non-agression qui mettent 100). Le Game Master ne fait qu'une passe de *validation finale*.
